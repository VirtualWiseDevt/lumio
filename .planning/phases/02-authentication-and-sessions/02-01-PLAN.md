<task type="auto">
  <name>Task 2: Env config, Express types, validators, and phone utils</name>
  <files>
    api/src/config/env.ts
    api/src/types/express.d.ts
    api/src/validators/auth.validators.ts

  <action>
    1. Update `api/src/config/env.ts` -- add JWT_SECRET to the envSchema:
       ```typescript
       JWT_SECRET: z.string().min(32, "JWT_SECRET must be at least 32 characters"),
       ```
       This is required because jose HS256 needs a secret >= 32 bytes (256 bits).

    2. Create `api/src/types/express.d.ts` -- augment Express Request:
       ```typescript
       import type { User } from "../../generated/prisma/client.js";

       declare global {
         namespace Express {
           interface Request {
             user?: User;
             sessionId?: string;
           }
         }
       }
       ```
       Ensure the tsconfig.json `include` pattern covers this file (it does: `src/**/*`).

    3. Create `api/src/validators/phone.utils.ts`:
       - Export `normalizeKenyanPhone(phone: string): string`
       - Handle three formats:
         - `07XXXXXXXX` (local) -> `+2547XXXXXXXX`
         - `+2547XXXXXXXX` (international with +) -> pass through
         - `2547XXXXXXXX` (international without +) -> prepend `+`
       - Strip spaces and hyphens first
       - Throw Error("Invalid Kenyan phone number") if none match
       - Each valid format must have exactly 10 digits (local) or 12 digits (international)

    4. Create `api/src/validators/auth.validators.ts`:
       Import `z` from "zod" and `normalizeKenyanPhone` from "./phone.utils.js".

       Export these Zod schemas:

       **registerSchema:**
       ```
       {
         name: z.string().min(2).max(100).trim(),
         email: z.string().email().toLowerCase().trim(),
         phone: z.string().transform(normalizeKenyanPhone),
         password: z.string()
           .min(8, "Password must be at least 8 characters")
           .regex(/[a-z]/, "Password must contain a lowercase letter")
           .regex(/[A-Z]/, "Password must contain an uppercase letter")
           .regex(/[0-9]/, "Password must contain a digit"),
         referralCode: z.string().min(1, "Referral code is required").trim(),
       }
       ```

       **loginSchema:**
       ```
       {
         email: z.string().email().toLowerCase().trim(),
         password: z.string().min(1),
       }
       ```

       **changePasswordSchema:**
       ```
       {
         currentPassword: z.string().min(1),
         newPassword: z.string()
           .min(8)
           .regex(/[a-z]/)
           .regex(/[A-Z]/)
           .regex(/[0-9]/),
       }
       ```

       **forgotPasswordSchema:**
       ```
       { email: z.string().email().toLowerCase().trim() }
       ```

       **resetPasswordSchema:**
       ```
       {
         token: z.string().min(1),
         newPassword: z.string()
           .min(8)
           .regex(/[a-z]/)
           .regex(/[A-Z]/)
           .regex(/[0-9]/),
       }
       ```

       IMPORTANT: The phone field in registerSchema uses `.transform()` which means Zod will run normalizeKenyanPhone and throw a ZodError if the phone is invalid. Wrap the transform properly so Zod catches the thrown error.

    5. Add `JWT_SECRET` to the `.env` file (or `.env.example` if it exists). Use a development value that is at least 32 characters:
       ```
       JWT_SECRET=dev-secret-key-must-be-at-least-32-chars-long
       ```
  </action>
