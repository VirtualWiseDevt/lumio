    **token.service.ts:**

    Import `SignJWT` and `jwtVerify` from "jose", `crypto` from "node:crypto", and `env` from "../config/env.js".

    1. Create the HS256 secret: `const secret = new TextEncoder().encode(env.JWT_SECRET);`

    2. `signToken(payload: { sub: string; sid: string }): Promise<string>`
       - Use `new SignJWT(payload).setProtectedHeader({ alg: "HS256" }).setIssuedAt().setExpirationTime("7d").sign(secret)`
       - Returns the JWT string

    3. `verifyToken(token: string): Promise<{ sub: string; sid: string; iat: number; exp: number }>`
       - Use `jwtVerify(token, secret, { algorithms: ["HS256"] })`
       - Return `payload as { sub: string; sid: string; iat: number; exp: number }`

    4. `generateResetToken(): string`
       - Return `crypto.randomBytes(32).toString("hex")` (64-char hex string)

    5. `hashResetToken(token: string): string`
       - Return `crypto.createHash("sha256").update(token).digest("hex")`
       - Used to store hashed reset token in DB (prevents DB-leak compromise)

    **session.service.ts:**

    Import `prisma` from "../config/database.js" and `UAParser` from "ua-parser-js".

    1. Helper `getDeviceInfo(userAgent: string): { deviceName: string; deviceType: string }`
       - Use `new UAParser(userAgent)` to parse
       - `deviceName = \`${browser.name || "Unknown"} on ${os.name || "Unknown"}\``
       - `deviceType = device.type || "desktop"`

    2. `createSession(params: { userId: string; userAgent: string; ipAddress: string }): Promise<Session>`
       - Call `getDeviceInfo(params.userAgent)`
       - Calculate `expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000)`
       - Create session with `prisma.session.create({ data: { userId, deviceName, deviceType, ipAddress, token: crypto.randomUUID(), expiresAt } })`
       - Import `crypto` from "node:crypto" for randomUUID
       - The `token` field stores a unique identifier (not the JWT itself)
       - Return the created session

    3. `enforceDeviceLimit(userId: string): Promise<{ allowed: boolean; devices: Array<{ id: string; deviceName: string; lastActiveAt: Date }> }>`
       - Query `prisma.session.findMany({ where: { userId, expiresAt: { gt: new Date() } }, select: { id: true, deviceName: true, lastActiveAt: true }, orderBy: { lastActiveAt: "desc" } })`
       - CRITICAL: Filter `expiresAt: { gt: new Date() }` to exclude expired sessions from the count
       - If `activeSessions.length >= 2`, return `{ allowed: false, devices: activeSessions }`
       - Otherwise return `{ allowed: true, devices: activeSessions }`

    4. `getUserSessions(userId: string): Promise<Array<{ id: string; deviceName: string; lastActiveAt: Date; createdAt: Date }>>`
       - Query active sessions (expiresAt > now), select id, deviceName, lastActiveAt, createdAt
       - Order by lastActiveAt desc

    5. `deleteSession(sessionId: string, userId: string): Promise<void>`
       - Delete where `{ id: sessionId, userId }` (userId check prevents deleting other users' sessions)

    6. `deleteOtherSessions(currentSessionId: string, userId: string): Promise<number>`
       - Delete where `{ userId, id: { not: currentSessionId } }`
       - Return `result.count`

    7. `cleanupStaleSessions(): Promise<number>`
       - Calculate cutoff: `new Date(Date.now() - 7 * 24 * 60 * 60 * 1000)`
       - Delete where `OR: [{ expiresAt: { lt: new Date() } }, { lastActiveAt: { lt: cutoff } }]`
       - Return `result.count`

    **auth.service.ts:**

    Import: `argon2` (import * as argon2 from "argon2"), `prisma` from "../config/database.js", `signToken`, `generateResetToken`, `hashResetToken` from "./token.service.js", `createSession`, `enforceDeviceLimit`, `deleteOtherSessions` from "./session.service.js".

    1. `register(data: { name: string; email: string; phone: string; password: string; referralCode: string }, meta: { userAgent: string; ipAddress: string })`
       - Validate referral code exists: `prisma.user.findUnique({ where: { referralCode: data.referralCode } })`
       - If not found, throw error with code "INVALID_REFERRAL_CODE" and message "Invalid referral code"
       - Check for existing user by email OR phone: `prisma.user.findFirst({ where: { OR: [{ email: data.email }, { phone: data.phone }] } })`
       - If exists, throw error with code "ACCOUNT_EXISTS" and message "Account already exists" (generic -- no field-specific error to prevent enumeration)
       - Hash password: `await argon2.hash(data.password)` (uses Argon2id with secure defaults)
       - Generate referral code for new user: `crypto.randomBytes(4).toString("hex")` (8-char alphanumeric)
       - Use a Prisma transaction to atomically:
         a. Create user with `{ name, email, phone, passwordHash, referralCode: generatedCode }`
         b. Create Referral record linking referrer to new user: `{ referrerId: referrer.id, refereeId: newUser.id }`
         c. Create session via `createSession({ userId: newUser.id, userAgent: meta.userAgent, ipAddress: meta.ipAddress })`
       - NOTE: The transaction needs to handle createSession carefully. Since createSession uses prisma directly, you may need to pass the transaction client ($transaction with interactive transaction) or create the session inside the transaction manually.
       - Use interactive transaction: `prisma.$transaction(async (tx) => { ... })` and create session record directly with `tx.session.create(...)` instead of calling createSession (to keep it in the same transaction).
       - After transaction, sign JWT: `await signToken({ sub: user.id, sid: session.id })`
       - Return `{ user: { id, name, email, phone, referralCode }, token: jwt }`

    2. `login(data: { email: string; password: string }, meta: { userAgent: string; ipAddress: string })`
       - Find user by email: `prisma.user.findUnique({ where: { email: data.email } })`
       - If not found, return error with code "INVALID_CREDENTIALS" and message "Invalid email or password"
       - Check account lock: if `user.lockedUntil && user.lockedUntil > new Date()`, return error with code "ACCOUNT_LOCKED" and message "Account is temporarily locked. Try again later."
       - Verify password: `await argon2.verify(user.passwordHash, data.password)`
       - If password wrong:
         - Increment failedLoginAttempts: `prisma.user.update({ where: { id: user.id }, data: { failedLoginAttempts: { increment: 1 } } })`
         - If updated failedLoginAttempts >= 5, lock account: `prisma.user.update({ where: { id: user.id }, data: { lockedUntil: new Date(Date.now() + 15 * 60 * 1000), failedLoginAttempts: 0 } })`
         - Return error with code "INVALID_CREDENTIALS" and message "Invalid email or password"
       - On success, reset failed attempts: `prisma.user.update({ where: { id: user.id }, data: { failedLoginAttempts: 0, lockedUntil: null } })`
       - Check device limit: `await enforceDeviceLimit(user.id)`
       - If not allowed, return `{ deviceLimitReached: true, devices: result.devices }` (HTTP 409 in route)
       - Create session: `await createSession({ userId: user.id, userAgent: meta.userAgent, ipAddress: meta.ipAddress })`
       - Sign JWT: `await signToken({ sub: user.id, sid: session.id })`
       - Return `{ user: { id, name, email, referralCode }, token: jwt }`

    3. `logout(sessionId: string, userId: string): Promise<void>`
       - Delete the session: `prisma.session.delete({ where: { id: sessionId, userId } })`
       - Wrap in try/catch -- if session not found, silently succeed (idempotent)

    4. `changePassword(userId: string, currentSessionId: string, data: { currentPassword: string; newPassword: string })`
       - Load user: `prisma.user.findUnique({ where: { id: userId } })`
       - Verify current password with argon2.verify
       - If wrong, throw error with code "INVALID_PASSWORD"
       - Hash new password: `await argon2.hash(data.newPassword)`
       - Update user passwordHash: `prisma.user.update({ where: { id: userId }, data: { passwordHash: newHash } })`
       - Delete all other sessions: `await deleteOtherSessions(currentSessionId, userId)`
       - Return `{ message: "Password changed successfully" }`

    5. `forgotPassword(email: string)`
       - Find user by email
       - If not found, return success anyway (prevent enumeration): `{ message: "If an account exists, a reset link has been sent" }`
       - Generate reset token: `generateResetToken()`
       - Hash it: `hashResetToken(token)`
       - Store hashed token and expiry (1 hour): `prisma.user.update({ where: { id: user.id }, data: { passwordResetToken: hashedToken, passwordResetExpiry: new Date(Date.now() + 60 * 60 * 1000) } })`
       - Log the raw token to console for development: `console.log(\`[DEV] Password reset token for ${email}: ${token}\`)`
       - NOTE: Email sending is a placeholder for this phase (console.log only). Real email service added in Phase 9.
       - Return `{ message: "If an account exists, a reset link has been sent" }`

    6. `resetPassword(token: string, newPassword: string)`
       - Hash incoming token: `hashResetToken(token)`
       - Find user with matching token AND non-expired: `prisma.user.findFirst({ where: { passwordResetToken: hashedToken, passwordResetExpiry: { gt: new Date() } } })`
       - If not found, throw error with code "INVALID_RESET_TOKEN"
       - Hash new password: `await argon2.hash(newPassword)`
       - Update user: `prisma.user.update({ where: { id: user.id }, data: { passwordHash: newHash, passwordResetToken: null, passwordResetExpiry: null, failedLoginAttempts: 0, lockedUntil: null } })`
       - Delete all user sessions: `prisma.session.deleteMany({ where: { userId: user.id } })`
       - Return `{ message: "Password has been reset. Please log in." }`

    Define a custom error class for auth errors:
    ```typescript
    export class AuthError extends Error {
      constructor(public code: string, message: string, public statusCode: number = 400) {
        super(message);
        this.name = "AuthError";
      }
    }
    ```

    Use AuthError throughout (e.g., `throw new AuthError("INVALID_CREDENTIALS", "Invalid email or password", 401)`).

    **auth.middleware.ts:**

    Import: `type { Request, Response, NextFunction }` from "express", `verifyToken` from "../services/token.service.js", `prisma` from "../config/database.js".

    `requireAuth(req: Request, res: Response, next: NextFunction): Promise<void>`
    - Extract Authorization header
    - If missing or doesn't start with "Bearer ", return 401: `{ error: { message: "Authentication required", code: "AUTH_REQUIRED" } }`
    - Extract token (slice(7))
    - Try verifyToken(token)
    - On catch, return 401: `{ error: { message: "Invalid token", code: "INVALID_TOKEN" } }`
    - Load session: `prisma.session.findUnique({ where: { id: payload.sid }, include: { user: true } })`
    - If no session or session.expiresAt < new Date(), return 401: `{ error: { message: "Session expired", code: "SESSION_EXPIRED" } }`
    - Fire-and-forget lastActiveAt update: `prisma.session.update({ where: { id: session.id }, data: { lastActiveAt: new Date() } }).catch(() => {})` -- non-blocking to avoid latency
    - Set `req.user = session.user` and `req.sessionId = session.id`
    - Call `next()`
    - Wrap the entire function body in try/catch, catch calls `next(error)` for unexpected errors (Express 5 pattern)
