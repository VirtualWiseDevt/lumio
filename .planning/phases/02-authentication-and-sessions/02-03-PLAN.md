    **auth.routes.ts:**

    Import: `Router` from "express", all auth service functions from "../services/auth.service.js", `requireAuth` from "../middleware/auth.middleware.js", all validator schemas from "../validators/auth.validators.js", `AuthError` from "../services/auth.service.js".

    Create `export const authRouter = Router()`.

    All route handlers must be async functions. Express 5 auto-catches async errors, but catch AuthError specifically to set the right status code.

    Create a helper to validate request body with Zod:
    ```typescript
    function validate<T>(schema: ZodSchema<T>, data: unknown): T {
      const result = schema.safeParse(data);
      if (!result.success) {
        const error = new AuthError("VALIDATION_ERROR", result.error.errors[0].message, 422);
        throw error;
      }
      return result.data;
    }
    ```

    **POST /register** (public):
    - Validate body with registerSchema
    - Call `auth.register(body, { userAgent: req.headers["user-agent"] || "", ipAddress: req.ip || "unknown" })`
    - Return 201: `{ user, token }`
    - Catch AuthError: if code is "ACCOUNT_EXISTS" return 409, if "INVALID_REFERRAL_CODE" return 400

    **POST /login** (public):
    - Validate body with loginSchema
    - Call `auth.login(body, { userAgent, ipAddress })`
    - If result has `deviceLimitReached: true`, return 409: `{ error: { message: "Maximum devices reached. Remove a device to continue.", code: "DEVICE_LIMIT_REACHED" }, devices: result.devices }`
    - Otherwise return 200: `{ user, token }`
    - Catch AuthError: if code is "INVALID_CREDENTIALS" return 401, if "ACCOUNT_LOCKED" return 423

    **POST /login/force** (public):
    - This endpoint allows login by removing a specified session first
    - Validate body: loginSchema extended with `removeSessionId: z.string().uuid()`
    - Verify credentials first (same as login)
    - Delete the specified session: call `deleteSession(removeSessionId, user.id)` from session service
    - Then create new session and return JWT
    - This enables the UX flow: user sees device list -> picks one to remove -> calls /login/force

    **POST /logout** (protected -- use requireAuth):
    - Call `auth.logout(req.sessionId!, req.user!.id)`
    - Return 200: `{ message: "Logged out successfully" }`

    **POST /change-password** (protected):
    - Validate body with changePasswordSchema
    - Call `auth.changePassword(req.user!.id, req.sessionId!, body)`
    - Return 200: `{ message: "Password changed successfully" }`
    - Catch AuthError: if code is "INVALID_PASSWORD" return 401

    **POST /forgot-password** (public):
    - Validate body with forgotPasswordSchema
    - Call `auth.forgotPassword(body.email)`
    - Always return 200: `{ message: "If an account exists, a reset link has been sent" }`

    **POST /reset-password** (public):
    - Validate body with resetPasswordSchema
    - Call `auth.resetPassword(body.token, body.newPassword)`
    - Return 200: `{ message: "Password has been reset. Please log in." }`
    - Catch AuthError: if code is "INVALID_RESET_TOKEN" return 400

    Add a centralized error handler at the bottom of the router (or wrap each handler):
    ```typescript
    // Wrap each async handler to catch AuthError and set status code
    // Express 5 catches async errors, but the global error handler needs the right status code
    ```
    Approach: In each route handler, wrap in try/catch. On AuthError, set `res.status(error.statusCode)` then throw (let Express 5 error middleware handle the response). Or respond directly in the catch block with `res.status(error.statusCode).json({ error: { message: error.message, code: error.code } })`.

    PREFER: Respond directly in catch to avoid double-sending. Pattern:
    ```typescript
    router.post("/register", async (req, res) => {
      try {
        const body = validate(registerSchema, req.body);
        const result = await register(body, { userAgent: req.headers["user-agent"] || "", ipAddress: req.ip || "unknown" });
        res.status(201).json(result);
      } catch (error) {
        if (error instanceof AuthError) {
          res.status(error.statusCode).json({ error: { message: error.message, code: error.code } });
          return;
        }
        throw error; // Re-throw unexpected errors for global handler
      }
    });
    ```

    **sessionCleanup.job.ts:**

    Import `cron` from "node-cron" and `cleanupStaleSessions` from "../services/session.service.js".

    ```typescript
    import cron from "node-cron";
    import { cleanupStaleSessions } from "../services/session.service.js";

    export function startSessionCleanupJob(): void {
      // Run every hour at minute 0
      cron.schedule("0 * * * *", async () => {
        try {
          const count = await cleanupStaleSessions();
          if (count > 0) {
            console.log(`[SESSION_CLEANUP] Removed ${count} stale sessions`);
          }
        } catch (error) {
          console.error("[SESSION_CLEANUP] Error:", error);
        }
      });
      console.log("[SESSION_CLEANUP] Scheduled hourly stale session cleanup");
    }
    ```

    Only log when sessions are actually removed (avoid noisy logs every hour when nothing to clean).
