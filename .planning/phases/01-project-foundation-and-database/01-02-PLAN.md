Create the configuration layer for the API:

**api/prisma.config.ts** (Prisma 7 configuration -- CRITICAL new pattern):
```typescript
import "dotenv/config";
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
  },
  datasource: {
    url: env("DATABASE_URL"),
  },
});
```
This file MUST be in the api/ root (not src/). Prisma 7 reads it automatically. See RESEARCH.md "Pattern 1: Prisma 7 Configuration".

**api/src/config/env.ts** (Zod-validated environment):
```typescript
import "dotenv/config";
import { z } from "zod";

const envSchema = z.object({
  NODE_ENV: z.enum(["development", "production", "test"]).default("development"),
  PORT: z.coerce.number().default(5000),
  DATABASE_URL: z.string().url(),
  CORS_ORIGINS: z
    .string()
    .transform((s) => s.split(","))
    .default("http://localhost:3000,http://localhost:3001"),
});

const parsed = envSchema.safeParse(process.env);

if (!parsed.success) {
  console.error("Invalid environment variables:", parsed.error.flatten().fieldErrors);
  process.exit(1);
}

export const env = parsed.data;
export type Env = z.infer<typeof envSchema>;
```
See RESEARCH.md "Pattern 4: Zod-Validated Environment Configuration". This fails fast on missing/invalid env vars.

NOTE: The `z.string().url()` validation for DATABASE_URL may reject `postgresql://` URLs depending on the Zod version. If it does, change to `z.string().min(1)` instead. The important thing is that DATABASE_URL is required and non-empty.

**api/src/config/database.ts** (Prisma client singleton):
```typescript
import "dotenv/config";
import { PrismaPg } from "@prisma/adapter-pg";
import { PrismaClient } from "../generated/prisma/client.js";

const adapter = new PrismaPg({
  connectionString: process.env.DATABASE_URL!,
});

const prisma = new PrismaClient({ adapter });

export { prisma };
```
CRITICAL: Import PrismaClient from the GENERATED path `"../generated/prisma/client.js"`, NOT from `"@prisma/client"`. The `.js` extension is REQUIRED for ESM. See RESEARCH.md "Pitfall 1: Prisma Import Path Confusion" and "Pitfall 2: Missing ESM File Extensions".

NOTE: This file will show TypeScript errors until `npx prisma generate` is run (which creates the generated client). This is expected. The generated/ directory is created in Plan 03 when the schema is written and generated.

All imports must use `.js` extensions even for `.ts` files (ESM requirement with NodeNext module resolution).

Create the Express 5 application layer:

**api/src/middleware/error.middleware.ts** (Global error handler):
- Export `errorHandler` function with 4 parameters: `(err: Error, req: Request, res: Response, next: NextFunction)`
- Log error with method and path: `console.error([ERROR] ${req.method} ${req.path}: err.message)`
- Set status code: use `res.statusCode` if not 200, otherwise 500
- Return JSON: `{ error: { message, stack? } }` -- only include stack in non-production
- Express 5 requires the 4-arg signature to recognize this as error middleware
- See RESEARCH.md "Pattern 3: Express 5 Error Handling"

**api/src/middleware/notFound.middleware.ts** (404 handler):
- Export `notFoundHandler` function with `(req: Request, res: Response)` signature
- Return 404 JSON: `{ error: { message: "Route ${req.method} ${req.path} not found" } }`
- See RESEARCH.md "404 Handler" code example

**api/src/routes/health.routes.ts** (Health check):
- Export `healthRouter` using `Router()`
- GET `/` handler (async): try `prisma.$queryRaw\`SELECT 1\`` then return `{ status: "ok", timestamp, database: "connected", uptime: process.uptime() }`
- On catch: return 503 with `{ status: "error", timestamp, database: "disconnected" }`
- Import prisma from `"../config/database.js"` (with .js extension)
- Express 5: No asyncHandler needed -- promise rejections are automatically caught
- See RESEARCH.md "Health Check Endpoint" code example

**api/src/routes/index.ts** (Route aggregator):
- Import and re-export healthRouter
- Export a function `registerRoutes(app: Express)` that mounts all routers:
  - `app.use("/health", healthRouter)`
  - Comment placeholders for future routes: `// app.use("/api/auth", authRouter)` etc.

**api/src/app.ts** (App factory):
- Export `buildApp()` function that creates and configures the Express app
- Apply middleware in order: helmet(), cors({ origin: env.CORS_ORIGINS, credentials: true }), express.json({ limit: "10mb" }), express.urlencoded({ extended: false }), rateLimit({ windowMs: 60000, max: 100, standardHeaders: true, legacyHeaders: false })
- Mount routes via registerRoutes or directly
- Mount notFoundHandler AFTER routes
- Mount errorHandler LAST (must be after notFoundHandler)
- Import env from `"./config/env.js"` (with .js extension)
- See RESEARCH.md "Pattern 2: Express 5 App Factory"

**api/src/server.ts** (HTTP server + graceful shutdown):
- `import "dotenv/config"` FIRST (before any other imports)
- Import buildApp from `"./app.js"`, env from `"./config/env.js"`, prisma from `"./config/database.js"`
- Create app via buildApp(), start listening on env.PORT
- Log: `API server running on port ${PORT}`
- Graceful shutdown handler: on SIGTERM and SIGINT, close server, disconnect prisma, exit 0
- See RESEARCH.md "Pattern 2: Express 5 App Factory" server.ts example

**api/src/types/express.d.ts** (Type augmentation placeholder):
- Create an empty augmentation file for future Express Request extensions (user property, etc.)
- For now, just export an empty module declaration or a comment explaining this will be used in Phase 2 for auth:
```typescript
// Express Request type augmentation
// Will be extended in Phase 2 to add user property to Request
export {};
```

CRITICAL Express 5 reminders (from RESEARCH.md):
- Do NOT use `express-async-handler` -- Express 5 catches async errors natively
- Do NOT use `res.send(body, status)` -- use `res.status(status).json(body)` instead
- `req.body` is `undefined` (not `{}`) when no parser matches -- guard against this in future routes
- Use `express.urlencoded({ extended: false })` -- extended is false by default in Express 5

All local imports MUST use `.js` extensions (e.g., `"./app.js"`, `"./config/env.js"`). This is required by Node.js ESM with `"type": "module"`.
