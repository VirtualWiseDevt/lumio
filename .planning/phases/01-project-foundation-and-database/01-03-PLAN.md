  <action>
Create the complete Prisma 7 schema at `api/prisma/schema.prisma` with ALL entities specified in RESEARCH.md "Complete Prisma Schema for Lumio". The schema must include:

**Generator block** (Prisma 7 format):
```prisma
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

datasource db {
  provider = "postgresql"
}
```
CRITICAL: `provider` is `"prisma-client"` (NOT `"prisma-client-js"`). The `output` field is REQUIRED. No `url` in datasource block (it lives in prisma.config.ts). See RESEARCH.md "Pattern 1".

**Enums** (5 total):
- `Role`: USER, ADMIN
- `ContentType`: MOVIE, SERIES, DOCUMENTARY, CHANNEL
- `SubscriptionStatus`: ACTIVE, EXPIRED, CANCELLED
- `PaymentStatus`: PENDING, SUCCESS, FAILED, EXPIRED
- `CouponType`: PERCENTAGE, FIXED_AMOUNT, FREE_DAYS

**Models** (14 total with all fields, relations, and indexes):

1. **User** - id (uuid), email (unique), phone (unique), passwordHash, name, role (default USER), isActive (default true), createdAt, updatedAt. Relations: sessions, payments, subscriptions, watchProgress, watchlist, favorites, referralsMade (ReferrerRelation), referredBy (RefereeRelation), activityLogs. Indexes: email, phone, role.

2. **Session** - id (uuid), userId, deviceName, deviceType, ipAddress, token (unique), expiresAt, lastActiveAt (default now), createdAt. Relation: user (cascade delete). Indexes: userId, expiresAt, token.

3. **Content** - id (uuid), type (ContentType), title, description?, releaseYear?, duration?, ageRating?, quality?, categories (String[]), posterPortrait?, posterLandscape?, trailerUrl?, videoUrl?, streamUrl?, matchScore?, isPublished (default false), isFeatured (default false), createdAt, updatedAt. Relations: seasons, watchProgress, watchlist, favorites. Indexes: type, isPublished, isFeatured, title, compound [type, isPublished].

4. **Season** - id (uuid), contentId, number, title?, createdAt. Relation: content (cascade delete), episodes. Unique: [contentId, number]. Index: contentId.

5. **Episode** - id (uuid), seasonId, number, title, description?, duration?, videoUrl?, thumbnail?, createdAt. Relation: season (cascade delete). Unique: [seasonId, number]. Index: seasonId.

6. **Plan** - id (uuid), name (unique), price (Int, KES), durationDays (Int), isActive (default true), createdAt, updatedAt. Relations: payments, subscriptions.

7. **Payment** - id (uuid), userId, planId, amount (Int), discount (Int, default 0), status (PaymentStatus default PENDING), method (default "MPESA"), checkoutRequestId? (unique), mpesaReceiptNumber?, phoneNumber?, resultCode?, resultDesc?, createdAt, updatedAt. Relations: user (cascade), plan. Indexes: userId, planId, status, checkoutRequestId, createdAt.

8. **Subscription** - id (uuid), userId, planId, status (SubscriptionStatus default ACTIVE), startsAt, expiresAt, autoRenew (default false), createdAt, updatedAt. Relations: user (cascade), plan. Indexes: userId, status, expiresAt, compound [userId, status].

9. **WatchProgress** - id (uuid), userId, contentId, episodeId?, timestamp (Int, seconds), duration (Int, seconds), completed (default false), updatedAt, createdAt. Relations: user (cascade), content (cascade). Unique: [userId, contentId, episodeId]. Indexes: userId, contentId, updatedAt.

10. **Watchlist** - id (uuid), userId, contentId, createdAt. Relations: user (cascade), content (cascade). Unique: [userId, contentId]. Index: userId.

11. **Favorite** - id (uuid), userId, contentId, createdAt. Relations: user (cascade), content (cascade). Unique: [userId, contentId]. Index: userId.

12. **Referral** - id (uuid), referrerId, refereeId (unique), creditAmount (Int, default 0), isRedeemed (default false), createdAt. Relations: referrer (ReferrerRelation, cascade), referee (RefereeRelation, cascade). Indexes: referrerId, refereeId.

13. **Coupon** - id (uuid), code (unique), type (CouponType), value (Int), maxUses?, currentUses (default 0), expiresAt?, isActive (default true), createdAt. Indexes: code, isActive.

14. **ActivityLog** - id (uuid), userId?, action, entityType, entityId?, details (Json?), ipAddress?, createdAt. Relation: user (SetNull on delete). Indexes: userId, action, entityType, createdAt.

Use the EXACT schema from RESEARCH.md "Complete Prisma Schema for Lumio" section. Copy it precisely -- it has been verified against the requirements.

After creating the schema file:

1. Start PostgreSQL with Docker: `docker compose up -d postgres` (just the database, not the full stack)
2. Wait for PostgreSQL to be healthy: `docker compose exec postgres pg_isready -U lumio -d lumio` (retry if needed)
3. Generate the Prisma client: `cd api && npx prisma generate` (this creates api/src/generated/prisma/)
4. Run the initial migration: `cd api && npx prisma migrate dev --name init` (this creates the migration files and applies them)

If `prisma migrate dev` fails because the database is not reachable via localhost (Docker networking), make sure the .env file has the correct DATABASE_URL for localhost access (port 5432 must be forwarded). The Docker Compose postgres service should have ports: "5432:5432".

After migration succeeds:
5. Verify the generated client exists: `ls api/src/generated/prisma/`
6. Verify migration was created: `ls api/prisma/migrations/`

**Schema design rationale** (from RESEARCH.md):
- UUIDs prevent enumeration attacks
- Single Content table with ContentType enum enables cross-type queries
- PostgreSQL String[] arrays for categories (simpler than junction table)
- Integer KES amounts for M-Pesa compatibility
- Composite unique constraints prevent duplicates
- Indexes on ALL foreign keys (Prisma does NOT auto-index FKs)
- Cascade deletes are appropriate for this domain

   `curl -s http://localhost:5000/nonexistent | jq .`

   ```json
   {

     }
   }
   ```

   `curl -s -I -X OPTIONS http://localhost:5000/health -H "Origin: http://localhost:3000" -H "Access-Control-Request-Method: GET"`
   Expected: Response includes `Access-Control-Allow-Origin` header

5. Test rate limiting headers:
   `curl -s -I http://localhost:5000/health`
   Expected: Response includes `RateLimit-*` or `X-RateLimit-*` headers

6. Test security headers (helmet):
   `curl -s -I http://localhost:5000/health`
   Expected: Response includes security headers like `X-Content-Type-Options`, `X-Frame-Options`, etc.

- Prisma client not generated: run `cd api && npx prisma generate`

- Import errors: ensure all imports use .js extensions and api/package.json has "type": "module"
