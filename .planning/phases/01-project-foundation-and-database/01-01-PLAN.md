Create the root monorepo structure with these exact files:

**package.json** (root):
- `"name": "lumio"`, `"private": true`
- `"workspaces": ["api", "client", "admin"]`
- Scripts: `dev:api`, `dev:client`, `dev:admin`, `docker:up`, `docker:down`, `db:migrate`, `db:generate`, `db:studio` (see RESEARCH.md "npm Workspaces Root package.json" for exact scripts)

**tsconfig.base.json** (root):
- `"target": "ES2023"`, `"module": "NodeNext"`, `"moduleResolution": "NodeNext"`
- `"strict": true`, `"esModuleInterop": true`, `"skipLibCheck": true`
- `"forceConsistentCasingInFileNames": true`, `"resolveJsonModule": true`
- `"declaration": true`, `"declarationMap": true`, `"sourceMap": true`
- `"verbatimModuleSyntax": true`

**docker-compose.yml**:
- PostgreSQL 16 Alpine service (`lumio-postgres`): ports 5432:5432, healthcheck with `pg_isready`, named volume `postgres_data`, credentials POSTGRES_DB=lumio, POSTGRES_USER=lumio, POSTGRES_PASSWORD=lumio_dev_password
- API service (`lumio-api`): build from `./api` using `Dockerfile.dev`, ports 5000:5000, depends_on postgres with `condition: service_healthy`, environment vars for NODE_ENV/PORT/DATABASE_URL/CORS_ORIGINS, volume mounts for `./api/src:/app/src`, `./api/prisma:/app/prisma`, `./api/prisma.config.ts:/app/prisma.config.ts`
- API command: `sh -c "npx prisma generate && npx prisma migrate deploy && npx tsx --watch src/server.ts"`
- DATABASE_URL inside Docker uses hostname `postgres` (not localhost): `postgresql://lumio:lumio_dev_password@postgres:5432/lumio?schema=public`
- See RESEARCH.md "Pattern 5: Docker Compose with Health Checks" for exact structure

**api/Dockerfile.dev**:
- FROM node:22-alpine, WORKDIR /app
- COPY package.json package-lock.json ./ then RUN npm ci
- COPY . .
- CMD ["npx", "tsx", "--watch", "src/server.ts"]

**.env** (for local dev outside Docker):
```
NODE_ENV=development
PORT=5000
DATABASE_URL=postgresql://lumio:lumio_dev_password@localhost:5432/lumio?schema=public
CORS_ORIGINS=http://localhost:3000,http://localhost:3001
```

**.env.example** (template without secrets):

- Dependencies: `express@5`, `cors`, `helmet`, `express-rate-limit`, `dotenv`, `zod@^3.24` (Zod 4 not yet stable enough -- use latest Zod 3.x which is the current stable), `@prisma/client`, `@prisma/adapter-pg`, `pg`
- Dev dependencies: `prisma`, `@types/pg`, `@types/express`, `@types/cors`, `typescript`, `tsx`, `@types/node`

NOTE on Zod version: The RESEARCH.md mentions zod 4.3.x but Zod 4 is brand new. Use `zod@^3.24` which is the proven stable version. The API (z.object, z.string, z.coerce, safeParse) is identical between Zod 3.24 and Zod 4 for our use case. If zod@^3.24 fails to install, fall back to just `zod` (latest).
