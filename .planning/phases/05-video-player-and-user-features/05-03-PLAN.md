  <action>
1. Create `client/src/stores/player.ts` — Zustand store following same pattern as stores/ui.ts:
   - All fields from PlayerState interface with setters
   - Initial state: isPlaying=false, isMuted=false, isFullscreen=false, isBuffering=false, currentTime=0, duration=0, buffered=0, volume=1, showControls=true, showNextEpisode=false, bufferingStartTime=null
   - Individual setters: setPlaying, setMuted, setFullscreen, setBuffering, setCurrentTime, setDuration, setBuffered, setVolume, setShowControls, setShowNextEpisode, setBufferingStartTime
   - `reset()` method to reset all state (called on unmount)

2. Create `client/src/api/progress.ts` — following same pattern as api/content.ts:
   - `saveProgress(data: ProgressData)`: POST /progress with body. Returns void.
   - `getProgress(contentId: string, episodeId?: string)`: GET /progress/:contentId with optional ?episodeId query. Returns { timestamp, duration, completed } | null.
   - `getContinueWatching(months?: number)`: GET /progress/continue-watching?months=N. Returns ContinueWatchingItem[].
   - All use `api` from ./client

3. Create `client/src/api/my-list.ts`:
   - `getMyList()`: GET /my-list. Returns MyListItem[].
   - `isInMyList(contentId: string)`: GET /my-list/:contentId. Returns boolean (from response.data.inList).
   - `addToMyList(contentId: string)`: POST /my-list/:contentId. Returns void.
   - `removeFromMyList(contentId: string)`: DELETE /my-list/:contentId. Returns void.

4. Create `client/src/api/user.ts`:
   - `getUserProfile()`: GET /user/profile. Returns UserProfile.
   - `updateUserProfile(data: { name?: string; phone?: string })`: PATCH /user/profile. Returns UserProfile.
   - `updatePreferences(data: { newsletter: boolean })`: PATCH /user/preferences. Returns void.
   - `getUserSubscription()`: GET /user/subscription. Returns UserSubscription.
   - `getUserSessions()`: GET /sessions. Returns DeviceSession[]. (Uses existing sessions endpoint from Phase 2)
   - `deleteSession(sessionId: string)`: DELETE /sessions/:sessionId. Returns void. (Uses existing sessions endpoint)
  </action>
