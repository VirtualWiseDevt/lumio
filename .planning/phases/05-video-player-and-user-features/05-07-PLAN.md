  <action>
1. Create `client/src/hooks/use-progress-tracking.ts`:
   - Accept: videoRef (RefObject<HTMLVideoElement>), contentId (string), episodeId (string | null)
   - 10-second interval: use setInterval in useEffect. Every 10s, read videoRef.current.currentTime and duration, call saveProgress({ contentId, episodeId, timestamp: Math.floor(currentTime), duration: Math.floor(duration) }) via the API client.
   - Save on pause: add "pause" event listener on video element, call saveProgress.
   - Save on unload: use visibilitychange + pagehide events (NOT beforeunload per RESEARCH.md).
     - On visibilitychange (document.visibilityState === "hidden"): build JSON blob and call navigator.sendBeacon("/api/progress", blob)
     - On pagehide: same sendBeacon call
     - sendBeacon body MUST be `new Blob([jsonString], { type: "application/json" })` (not plain string -- Express won't parse text/plain)
   - Store interval ID in useRef, clear on unmount
   - Clean up all event listeners on unmount
   - Detect 90% completion: when currentTime/duration >= 0.9 and onComplete callback is provided, call it (used for next-episode trigger). Use a ref flag (hasTriggeredComplete) to prevent multiple calls.

2. Create `client/src/components/player/NextEpisodeOverlay.tsx`:
   - "use client" directive
   - Props: nextEpisode ({ title, thumbnail, seasonNumber, episodeNumber } | null), onPlay (() => void), onCancel (() => void)
   - 10-second countdown timer using useEffect + setInterval (1s ticks)
   - When countdown reaches 0, call onPlay()
   - Display: semi-transparent overlay in bottom-right area of player
     - "Next Episode" heading
     - Episode thumbnail (small, from mediaUrl), title, "S{N}:E{N}" label
     - Countdown ring or text: "Playing in {N}..."
     - "Play Now" button and "Cancel" button
   - Cancel button stops the countdown and hides the overlay
   - Use isAdvancing ref to prevent race conditions (per RESEARCH.md Pitfall 7)
  </action>
