1. Create admin/src/api/video-upload.ts:
   - Import apiClient from "./client"
   - getPresignedUploadUrl(data: { contentId: string; episodeId?: string; filename: string; contentType: string; fileSize: number }): Promise<{ uploadUrl: string; key: string }>
     - POST /api/admin/video-upload/presign with data
   - confirmVideoUpload(data: { contentId: string; episodeId?: string; key: string }): Promise<{ success: boolean; key: string }>
     - POST /api/admin/video-upload/confirm with data

2. Create admin/src/components/content/VideoUploader.tsx:
   Props: { contentId: string; episodeId?: string; currentKey?: string | null; transcodingStatus?: string | null; onUploadComplete?: (key: string) => void }

   Component behavior:
   - Shows a file input accepting video/mp4,video/x-matroska,video/quicktime
   - Max file size: 5GB (validate before upload)
   - Upload flow:
     a. User selects file
     b. Call getPresignedUploadUrl with file metadata
     c. PUT file directly to R2 using the presigned URL (fetch with method: PUT, body: file, headers: { "Content-Type": contentType })
     d. Call confirmVideoUpload to validate and store key
     e. Call onUploadComplete callback
   - States: idle (no file) | uploading | uploaded | error
   - Show: filename, file size, upload state text
   - If currentKey exists, show "Video uploaded" with the key filename
   - If uploading, show "Uploading..." text (no progress bar -- simple state indicator per context)
   - Style with Tailwind: border-dashed border, p-4, rounded, dark theme compatible

3. Create admin/src/components/content/TranscodingBadge.tsx:
   Props: { status: string | null | undefined; error?: string | null }

   Renders a Badge (shadcn) with:
   - null/undefined: no badge shown
   - "pending": yellow "Pending" badge
   - "processing": blue "Processing" badge with pulse animation
   - "completed": green "Ready" badge
   - "failed": red "Failed" badge, with tooltip or title showing error message
