  <files>docker-compose.yml, api/Dockerfile.dev, api/package.json</files>
  <action>
1. Add Redis service to docker-compose.yml:
   - image: redis:7-alpine
   - container_name: lumio-redis
   - ports: 6379:6379
   - healthcheck: redis-cli ping
   - Add to api service depends_on with condition: service_healthy
   - Add REDIS_HOST=redis and REDIS_PORT=6379 to api environment

2. Update api/Dockerfile.dev:
   - Add `RUN apk add --no-cache ffmpeg` after the FROM line (before WORKDIR)
   - This provides both ffmpeg and ffprobe binaries

3. Install npm dependencies in api workspace:
   - bullmq ^5.x (job queue)
   - ioredis ^5.x (Redis client, required by BullMQ)
   - @aws-sdk/client-s3 ^3.x (R2 storage via S3 API)
   - @aws-sdk/s3-request-presigner ^3.x (presigned URL generation)

   Run: npm install -w api bullmq ioredis @aws-sdk/client-s3 @aws-sdk/s3-request-presigner

4. Add R2 env vars to api/src/config/env.ts validation schema:
   - R2_ACCOUNT_ID: z.string().min(1)
   - R2_ACCESS_KEY_ID: z.string().min(1)
   - R2_SECRET_ACCESS_KEY: z.string().min(1)
   - R2_BUCKET_NAME: z.string().min(1)
   - REDIS_HOST: z.string().default("localhost")
   - REDIS_PORT: z.coerce.number().default(6379)

   Also add these to docker-compose.yml api environment with placeholder values for R2 (empty strings are fine for now since R2 is only used at runtime when actually uploading/transcoding).
   Actually, since env validation uses .min(1), use placeholder values like "placeholder" for R2 vars in docker-compose to allow the API to start. These will be replaced with real values when R2 is configured.

5. Rebuild containers: docker compose build api && docker compose up -d

<task type="auto">
  <name>Task 2: R2 client, Redis client, schema migration, and type definitions</name>
  <files>api/src/config/r2.ts, api/src/config/redis.ts, api/prisma/schema.prisma, api/src/types/transcode.types.ts</files>
  <action>
1. Create api/src/config/r2.ts:
   - Import S3Client from @aws-sdk/client-s3
   - Import env from ./env.js
   - Create and export r2Client = new S3Client with:
     - region: "auto"
     - endpoint: `https://${env.R2_ACCOUNT_ID}.r2.cloudflarestorage.com`
     - credentials: { accessKeyId: env.R2_ACCESS_KEY_ID, secretAccessKey: env.R2_SECRET_ACCESS_KEY }
   - Export R2_BUCKET_NAME = env.R2_BUCKET_NAME

2. Create api/src/config/redis.ts:
   - Import IORedis from "ioredis" (default import)
   - Import env from ./env.js
   - CRITICAL: Set maxRetriesPerRequest: null (BullMQ requirement)
   - Export redis = new IORedis({ host: env.REDIS_HOST, port: env.REDIS_PORT, maxRetriesPerRequest: null })
   - Add error handler: redis.on("error", (err) => console.error("Redis connection error:", err))

3. Add transcoding fields to Prisma schema on Content model:
   - transcodingStatus String? (values: "pending", "processing", "completed", "failed")
   - transcodingError  String?
   - sourceVideoKey    String? (R2 key for raw upload, e.g., "videos/{id}/raw/source.mp4")
   - hlsKey            String? (R2 prefix for HLS output, e.g., "videos/{id}/hls/")

   Add same 4 fields to Episode model.

   Create enum TranscodingStatus with values: PENDING, PROCESSING, COMPLETED, FAILED
   Actually, use a string field (not Prisma enum) to avoid migration complexity. Store as lowercase strings.

4. Run migration:
   - Write migration SQL manually (prisma migrate dev fails in non-interactive terminal)
   - Create migration: npx prisma migrate dev --name add_transcoding_fields --create-only
   - If that fails, create the SQL file manually in api/prisma/migrations/ with timestamp folder
   - Run: npx prisma migrate deploy
   - Run: npx prisma generate

5. Create api/src/types/transcode.types.ts:
   ```typescript
   export type TranscodingStatus = "pending" | "processing" | "completed" | "failed";

   export interface QualityPreset {
     name: string;       // "360p", "480p", "720p", "1080p"
     width: number;
     height: number;
     videoBitrate: string;  // e.g., "800k"
     audioBitrate: string;  // e.g., "96k"
     maxRate: string;
     bufSize: string;
   }

   export interface TranscodeJobData {
     contentId: string;
     episodeId?: string;
     sourceKey: string;    // R2 key of raw file
     outputPrefix: string; // R2 prefix for HLS output
   }

   export interface TranscodeResult {
     qualities: string[];
     masterPlaylistKey: string;
     duration: number;
   }

   export const QUALITY_PRESETS: QualityPreset[] = [
     { name: "360p", width: 640, height: 360, videoBitrate: "800k", audioBitrate: "96k", maxRate: "856k", bufSize: "1200k" },
     { name: "480p", width: 854, height: 480, videoBitrate: "1400k", audioBitrate: "128k", maxRate: "1498k", bufSize: "2100k" },
     { name: "720p", width: 1280, height: 720, videoBitrate: "2800k", audioBitrate: "128k", maxRate: "2996k", bufSize: "4200k" },
     { name: "1080p", width: 1920, height: 1080, videoBitrate: "5000k", audioBitrate: "192k", maxRate: "5350k", bufSize: "7500k" },
   ];
   ```
