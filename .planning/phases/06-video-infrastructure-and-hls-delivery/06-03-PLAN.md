  <name>Task 1: FFmpeg transcode service</name>
  <files>api/src/services/transcode.service.ts</files>
  <action>
Create api/src/services/transcode.service.ts with these functions:

1. probeSource(filePath: string): Promise<{ width: number; height: number; duration: number; fps: number; codec: string }>
   - Spawn ffprobe with args: ["-v", "quiet", "-print_format", "json", "-show_format", "-show_streams", filePath]
   - Parse JSON output, extract video stream info
   - Calculate fps from r_frame_rate (e.g., "24000/1001" -> 23.976 -> round to nearest int)
   - Return dimensions, duration (from format.duration), fps, codec

2. transcodeToHls(sourcePath: string, outputDir: string, preset: QualityPreset, segmentDuration: number, fps: number): Promise<void>
   - Skip if source height < preset.height (don't upscale -- e.g., 720p source skips 1080p)
   - Calculate GOP size: segmentDuration * fps (e.g., 4 * 24 = 96)
   - Spawn ffmpeg with these args:
     -i sourcePath
     -vf scale=${preset.width}:${preset.height}
     -c:v libx264 -preset medium -profile:v main -level 4.0
     -b:v ${preset.videoBitrate} -maxrate ${preset.maxRate} -bufsize ${preset.bufSize}
     -c:a aac -b:a ${preset.audioBitrate} -ar 48000 -ac 2
     -g ${gop} -keyint_min ${gop} -sc_threshold 0
     -hls_time ${segmentDuration} -hls_playlist_type vod
     -hls_segment_filename ${outputDir}/${preset.name}/segment%03d.ts
     ${outputDir}/${preset.name}/playlist.m3u8
   - CRITICAL keyframe flags: -g, -keyint_min, and -sc_threshold 0 must all be set identically across qualities
   - Create output directory before spawning: mkdir -p ${outputDir}/${preset.name}
   - Return promise that resolves on process exit code 0, rejects on non-zero
   - Pipe stderr to a buffer for error reporting (not stdout -- ffmpeg outputs to stderr)

3. generateMasterPlaylist(qualities: Array<{ preset: QualityPreset; playlistPath: string }>): string
   - Generate #EXTM3U master playlist content string
   - For each quality, add:
     #EXT-X-STREAM-INF:BANDWIDTH=${bandwidth},RESOLUTION=${width}x${height},CODECS="avc1.4d401f,mp4a.40.2"
     ${relative_path_to_playlist}
   - Bandwidth = parseInt(videoBitrate) * 1000 + parseInt(audioBitrate) * 1000
   - Actually, parse bitrate strings: "800k" -> 800000, "1400k" -> 1400000
   - Return the playlist content as a string

Helper: spawnAsync(command: string, args: string[]): Promise<{ stdout: string; stderr: string }>
   - Wraps child_process.spawn in a Promise
   - Collects stdout and stderr
   - Resolves on exit code 0, rejects with stderr on non-zero

Import spawn from "node:child_process".
Import mkdir from "node:fs/promises".
Import { QualityPreset, QUALITY_PRESETS } from "../types/transcode.types.js".
Use ESM imports with .js extensions.

Segment duration: Use 4 seconds (good balance of seek granularity vs segment count).

Create api/src/jobs/ directory and api/src/jobs/transcode.job.ts:

1. Create BullMQ Queue:

   - Export transcodeQueue

2. Export enqueueTranscode(data: TranscodeJobData): Promise<void>
   - Add job to queue: transcodeQueue.add("transcode", data, { attempts: 2, backoff: { type: "fixed", delay: 30000 } })
   - Update DB: set transcodingStatus = "pending" on Content or Episode

3. Create processTranscodeJob(job: Job<TranscodeJobData>): Promise<void>
   - This is the main processor function
   - Steps:
     a. Update DB: transcodingStatus = "processing"
     b. Create temp directory: /tmp/transcode-${job.id}
     c. Download source from R2 to temp dir:
        - Use GetObjectCommand to stream the file to disk
        - Import { Readable } from "node:stream"; import { pipeline } from "node:stream/promises"
        - import { createWriteStream } from "node:fs"
     d. Run probeSource on downloaded file to get dimensions, fps, duration
     e. Filter QUALITY_PRESETS to exclude presets where height > source height (no upscaling)
     f. For each applicable preset (sequentially, not parallel -- single server):
        - Call transcodeToHls(sourcePath, outputDir, preset, 4, fps)
        - Upload all generated .ts segments to R2: videos/{contentId}/hls/{preset.name}/segmentNNN.ts
        - Upload playlist.m3u8 to R2: videos/{contentId}/hls/{preset.name}/playlist.m3u8
        - Use readdir to list generated files, upload each with uploadToR2
     g. Generate master playlist string with generateMasterPlaylist
     h. Upload master.m3u8 to R2: videos/{contentId}/hls/master.m3u8
     i. Update DB: transcodingStatus = "completed", hlsKey = "videos/{contentId}/hls/"
     j. Clean up: rm -rf temp directory (in finally block)
   - On error:
     - Update DB: transcodingStatus = "failed", transcodingError = error.message
     - Re-throw to trigger BullMQ retry

4. Export startTranscodeWorker(): Worker
   - const worker = new Worker<TranscodeJobData>("transcode", processTranscodeJob, { connection: redis, concurrency: 1 })
   - Add event listeners: worker.on("completed"), worker.on("failed") for logging
   - Return worker
