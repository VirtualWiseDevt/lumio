1. Add helper to client/src/api/content.ts:
   ```
   export function getStreamUrl(contentId: string, episodeId?: string): string {
     const base = `/api/stream/${contentId}`;
     return episodeId ? `${base}?episode=${episodeId}` : base;
   }
   ```
   This returns a relative URL that Next.js will proxy to the API (or the client can use absolute URL to API).

   Actually, check how the client API base URL works. The client likely has an API_URL or uses Next.js rewrites. Look at client/src/api/client.ts for the base URL pattern.

   If the client uses an axios instance with baseURL, the stream URL should use the same base. But HLS.js needs a full URL it can fetch directly. So:
   - Export getStreamUrl that returns the full URL: `${API_BASE_URL}/api/stream/${contentId}${episodeId ? '?episode=' + episodeId : ''}`
   - The API_BASE_URL should be the same one used by the axios client (likely from env or hardcoded localhost:5000)

2. Update client/src/hooks/use-hls.ts:
   - The hook currently accepts a source URL and creates an HLS instance
   - Modify it so when the source is an m3u8 URL (or stream endpoint), it passes the URL to hls.js
   - hls.js needs the auth token for the stream endpoint (it's protected by requireAuth)
   - Configure hls.js with xhrSetup to add Authorization header:
     ```
     const hls = new Hls({
       xhrSetup: (xhr) => {
         const token = localStorage.getItem("token");
         if (token) {
           xhr.setRequestHeader("Authorization", `Bearer ${token}`);
         }
       },
     });
     ```
   - This ensures both playlist fetches and segment fetches include the JWT
   - Wait -- segment URLs are presigned R2 URLs that don't need auth. But playlist fetches go to the API which DOES need auth. hls.js will use xhrSetup for playlist fetches but segment fetches go directly to R2 presigned URLs which don't need headers. This should work fine since the presigned URL includes auth in the query string.

3. Update client/src/app/watch/[id]/page.tsx:
   - Import getStreamUrl from api/content
   - When determining the video source URL:
     - If content has transcodingStatus === "completed" (or hlsKey exists), use getStreamUrl(contentId, episodeId)
     - Otherwise fall back to the existing videoUrl/streamUrl behavior (for non-transcoded content or test files)
   - This graceful fallback ensures existing content without transcoding still works

4. The content detail type may need updating to include transcodingStatus and hlsKey.
