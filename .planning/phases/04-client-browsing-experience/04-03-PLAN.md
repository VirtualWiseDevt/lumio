Create a hook that manages the hero banner state machine:

```typescript
"use client";

interface UseHeroBannerOptions {
  itemCount: number;
  interval?: number; // default 8000ms
}

interface UseHeroBannerReturn {
  currentIndex: number;
  setCurrentIndex: (index: number) => void;
  isPaused: boolean;
  pause: () => void;
  resume: () => void;
}
```

Implementation:
- Auto-rotates through slides every `interval` ms (default 8000)
- `setCurrentIndex` resets the timer (user clicked a dot)
- `pause()` / `resume()` for when video is playing or user hovers
- Timer stops when component unmounts (cleanup in useEffect)
- When `isPaused` is true, auto-rotation stops
- Wraps around: after last slide, goes back to index 0

**2. `client/src/components/hero/HeroSlide.tsx`:**
Props: `{ content: Content, isActive: boolean, isMuted: boolean, isHeroVisible: boolean }`

Implements the video crossfade sequence from CONTEXT:
1. Render `posterLandscape` as backdrop image using Next.js `<Image>` (fill, priority for first slide). Use `mediaUrl()` helper to build the src. If no posterLandscape, use a dark placeholder gradient.
2. If `trailerUrl` exists and `isActive` is true: after ~1.5s delay, create a `<video>` element (hidden, muted, playsInline, loop) and start loading.
3. When video fires `canplay`, crossfade from image to video (opacity transition, ~500ms).
4. If `!isActive` or `!isHeroVisible`: pause the video, show the image.
5. If video fails to load or autoplay is blocked: stay on static image.
6. State machine: `"image" | "buffering" | "video"` -- use local state.
7. Video element: absolute positioned behind/over the image, controlled via opacity (0 during image phase, 1 during video phase, transition-opacity duration-500).

- Smooth transition on dot size change
