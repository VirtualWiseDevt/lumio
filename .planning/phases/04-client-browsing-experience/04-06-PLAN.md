**1. `client/src/hooks/use-search.ts`:**
`"use client"` hook.

```typescript
interface UseSearchReturn {
  query: string;
  debouncedQuery: string;
  setQuery: (q: string) => void;
  clear: () => void;
}
```

Implementation:
- Local state for `query` (immediate) and `debouncedQuery` (delayed 300ms)
- Use `useEffect` with setTimeout for debouncing (clear on cleanup)
- `clear()` resets both to empty string
- Also registers Ctrl+K / Cmd+K keyboard shortcut to open search (calls `useUIStore.openSearch`)
- Registers Escape key to close search (calls `useUIStore.closeSearch`)
- Keyboard listeners added/removed on mount/unmount

**2. `client/src/components/search/SearchOverlay.tsx`:**
`"use client"` component. No props (reads state from Zustand store).

- Reads `isSearchOpen` from `useUIStore`
- If not open: return null (or use AnimatePresence for exit animation)
- Full-screen fixed overlay: bg-background/95 backdrop-blur-sm, z-[60] (above detail modal), inset-0
- Top section:
  - Search input: large (text-2xl), auto-focused on mount, full-width, bg-transparent, border-b border-border
  - Search icon (lucide Search) left of input
  - X close button at top-right
  - Placeholder: "Search movies, series, documentaries..."
- Uses `useSearch` hook for query management
- Uses TanStack Query with `debouncedQuery`:
  - `queryKey: ["search", debouncedQuery]`
  - `queryFn: () => searchContent(debouncedQuery)`
  - `enabled: debouncedQuery.length >= 1`
- Results area: scrollable, px-[4%], mt-4
  - Renders `<SearchResults>` when data exists
  - Loading state: skeleton lines
  - Empty query state: "Start typing to search..." (text-muted, centered)
  - No results state: "No results found for '{query}'" (text-muted)
- Clicking a result closes the overlay and navigates to /title/{id}
- Use Motion AnimatePresence for overlay animation (opacity + slight translateY)
