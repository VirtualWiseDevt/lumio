Add a BrowseRow model to the Prisma schema:

```prisma
model BrowseRow {
  id         String   @id @default(uuid())
  title      String
  slug       String   @unique
  position   Int      @default(0)
  contentIds String[]
  isActive   Boolean  @default(true)
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt

  @@index([position])
  @@index([isActive])
}
```

This model lets admins configure custom home page rows (e.g., "Staff Picks", "Kenyan Originals") by storing an ordered list of content IDs.

<task type="auto">
  <name>Task 2: Create browse service with all data fetching functions</name>
  <files>api/src/services/browse.service.ts</files>
  <action>
Create `api/src/services/browse.service.ts` with these exports:

1. **getHomePageData()** - Returns `{ featured: Content[], rows: { title: string, slug: string, items: Content[] }[] }`
   - `featured`: Up to 5 published+featured content items (any type), ordered by updatedAt desc. Include all fields needed for hero banner (title, description, posterLandscape as backdrop, trailerUrl).
   - `rows`: Query active BrowseRow records ordered by position. For each row, fetch the content items by IDs (only published ones). Filter out any IDs that don't resolve to published content.

2. **getBrowsePageData(type: ContentType)** - Returns same shape as home.
   - `featured`: Up to 5 published+featured items of the given type.
   - `rows`: Group remaining published content of that type by category. For each category that has content of this type, create a row. Title is the category name. Items are the content with that category, ordered by createdAt desc, limited to 20 per row. Also include a "Recently Added" row with the 20 most recent published items of this type.

3. **getLiveTvData()** - Returns `{ categories: { name: string, channels: Content[] }[] }`
   - Get all published CHANNEL content. Group by categories (each channel can have multiple categories; list the channel under each). Each category group has a name and array of channels.

4. **getTitleDetail(id: string)** - Returns full content with seasons and episodes if SERIES.
   - For SERIES: include seasons (ordered by number) with their episodes (ordered by number).
   - For all types: return the full content record.
   - Return null if not found or not published.

5. **getSimilarTitles(id: string)** - Returns up to 6 published titles of the same type sharing the most categories.
   - First get the target content's type and categories.
   - Query published content of same type (excluding the current ID).
   - Sort by number of shared categories (application-level: fetch candidates, score by overlapping categories, take top 6).
   - If fewer than 6 matches by category, fill with recent content of same type.

6. **searchContent(query: string)** - Returns `{ movies: Content[], series: Content[], documentaries: Content[], channels: Content[] }`
   - Search by title (case-insensitive contains) across all published content.
   - Group results by ContentType.
   - Limit to 10 per type.

IMPORTANT:
- All queries filter on `isPublished: true`.
- Select only fields needed by the client (avoid returning videoUrl, streamUrl -- those are for the player phase).
- Use PrismaClient from `../config/database.js` (existing import pattern).
- Use .js extensions on all imports.
