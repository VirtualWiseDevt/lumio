     z.object({
       Body: z.object({
         stkCallback: z.object({
           MerchantRequestID: z.string(),
           CheckoutRequestID: z.string(),
           ResultCode: z.number(),
           ResultDesc: z.string(),
           CallbackMetadata: z.object({
             Item: z.array(z.object({
               Name: z.string(),
               Value: z.union([z.string(), z.number()]).optional(),
             })),
           }).optional(),
         }),
       }),
     })

2. Create api/src/services/payment.service.ts with these exports:

   `initiatePayment(userId: string, planId: string, phone: string): Promise<{ paymentId: string; checkoutRequestId: string; isExisting: boolean }>`
   - First check for existing PENDING payment for this user (double-payment protection layer 2)
   - If found, return { paymentId: existing.id, checkoutRequestId: existing.checkoutRequestId, isExisting: true }
   - If not found:
     - Look up the plan by ID (throw 404 if not found or inactive)
     - Normalize phone with normalizePhoneForDaraja
     - Generate idempotencyKey with crypto.randomUUID()
     - Create Payment record with status PENDING, idempotencyKey, phoneNumber, planId, userId, amount from plan.price
     - Call getMpesaClient().initiateSTKPush({ phone, amount: plan.price, accountRef: plan.name })
     - Update payment with checkoutRequestId from STK Push response
     - Return { paymentId, checkoutRequestId, isExisting: false }
   - Wrap in try/catch: if STK Push fails, update payment to FAILED with error details

   `processCallback(callbackData: parsed callback type): Promise<void>`
   - Extract CheckoutRequestID and ResultCode from callback
   - Find payment by checkoutRequestId WHERE status = "PENDING" (idempotency -- already-processed payments are skipped)
   - If no PENDING payment found, log and return (duplicate callback)
   - If ResultCode === 0 (success):
     - Extract metadata: amount, receiptNumber, transactionDate, phoneNumber
     - Call activateSubscription (from subscription.service.ts) within a Prisma $transaction:
       - Update payment: status SUCCESS, mpesaReceiptNumber, resultCode, resultDesc, rawCallback (full JSON)
       - Create subscription: userId, planId, status ACTIVE, startsAt now, expiresAt now + plan.durationDays
     - Use `prisma.$transaction(async (tx) => { ... })` for atomicity
   - If ResultCode !== 0 (failure):
     - Update payment: status FAILED, resultCode, resultDesc, rawCallback

   `getPaymentStatus(paymentId: string, userId: string): Promise<{ status, mpesaReceiptNumber?, planName? }>`
   - Find payment by id and userId (authorization check)
   - Return status, receipt number if success, plan name

   `getPaymentHistory(userId: string, page: number, limit: number): Promise<{ payments, total, page, totalPages }>`
   - Query payments for userId, ordered by createdAt desc
   - Include plan relation for plan name
   - Return paginated result with: id, amount, status, method, mpesaReceiptNumber, phoneNumber, createdAt, plan.name

   Helper: `extractCallbackMetadata(items)` to pull Amount, MpesaReceiptNumber, TransactionDate, PhoneNumber from callback Item array.

All imports use .js extensions. Use `import type` for type-only imports.

   - Accepts a Prisma transaction client (from $transaction in payment service)
   - Fetch the payment with plan relation using tx
   - Calculate expiresAt: check if user has an existing ACTIVE subscription with expiresAt > now
     - If yes: extend from existing expiresAt (subscription stacking)
     - If no: start from now
   - Create new subscription record: userId, planId, status ACTIVE, startsAt, expiresAt
   - If extending, update old subscription status to EXPIRED (replaced by new one)

   `hasActiveSubscription(userId: string): Promise<boolean>`
   - Query subscription where userId, status ACTIVE, expiresAt > new Date()
   - Return boolean

   `getActiveSubscription(userId: string): Promise<subscription with plan | null>`
