<task type="auto">
  <name>Task 1: Install deps, configure Multer and Sharp, create upload pipeline</name>
  <files>
    api/package.json
    api/src/config/upload.ts
    api/src/middleware/upload.middleware.ts
    api/src/services/upload.service.ts
  </files>
  <action>
    1. Install dependencies from api/:
       `npm install multer sharp`
       `npm install -D @types/multer`
       Note: If sharp install fails on Windows, run `npm rebuild sharp`.

    2. Create api/src/config/upload.ts:
       - Export UPLOAD_DIR: path.resolve(process.cwd(), 'uploads') -- relative to api/ working directory
       - Export size presets as config object:
         ```
         export const IMAGE_SIZES = {
           poster: {
             large: { width: 800, suffix: 'large' },
             medium: { width: 400, suffix: 'medium' },
             thumbnail: { width: 200, suffix: 'thumbnail' },
           },
           backdrop: {
             large: { width: 1920, suffix: 'large' },
             medium: { width: 960, suffix: 'medium' },
           },
         } as const;
         ```
       - Export ALLOWED_MIME_TYPES: ['image/jpeg', 'image/png', 'image/webp']
       - Export MAX_FILE_SIZE: 10 * 1024 * 1024 (10MB)
       - Export WEBP_QUALITY: { original: 90, resized: 80 }
       - Export a function `ensureUploadDirs()` that creates all needed subdirectories:
         uploads/posters/original, uploads/posters/large, uploads/posters/medium, uploads/posters/thumbnail
         uploads/backdrops/original, uploads/backdrops/large, uploads/backdrops/medium
         Use fs.mkdirSync with { recursive: true }

    3. Create api/src/middleware/upload.middleware.ts:
       - Configure Multer with memoryStorage (buffer stays in memory for Sharp processing)
       - Set fileFilter to reject non-allowed MIME types
       - Set limits.fileSize to MAX_FILE_SIZE
       - Export `imageUpload` as the configured Multer instance
       - Handle Multer errors: if MulterError with code LIMIT_FILE_SIZE, return 413. Otherwise 400.

    4. Create api/src/services/upload.service.ts:
       - Import sharp from 'sharp'
       - Import crypto for randomUUID
       - Import config from upload.ts

       `processImage(buffer: Buffer, type: 'poster' | 'backdrop')`:
       - Generate UUID for filename
       - Get size presets for the type from IMAGE_SIZES
       - For each size preset: sharp(buffer).resize(width, null, { fit: 'inside', withoutEnlargement: true }).webp({ quality: WEBP_QUALITY.resized }).toFile(outputPath)
       - Also save original as WebP: sharp(buffer).webp({ quality: WEBP_QUALITY.original }).toFile(origPath)
       - Store files at: uploads/{type}s/{size}/{uuid}-{size}.webp
       - Return object with relative paths for each size: { original: 'posters/original/uuid-original.webp', large: 'posters/large/uuid-large.webp', ... }
       - Paths are RELATIVE (no leading slash, no 'uploads/' prefix) -- the media route resolves them

       `deleteImageSet(paths: Record<string, string>)`:
       - For each path in the set, delete the file from uploads/ directory
       - Use fs.unlink with .catch(() => {}) to ignore missing files
       - Used when content is deleted or image is replaced

    Call ensureUploadDirs() when upload service is first used (or at app startup).
  </action>
  <verify>
    `cd api && npx tsc --noEmit` compiles clean.
    Verify sharp is importable: `cd api && node -e "import('sharp').then(s => console.log('Sharp OK:', s.default.versions))"`
  </verify>
  <done>Multer configured with memory storage and file validation. Sharp pipeline generates WebP images at multiple sizes. Upload directory structure auto-created. Size presets configurable.</done>
</task>

  <action>
    1. Create api/src/routes/upload.routes.ts:
       - Apply requireAuth, requireAdmin to all routes
       - POST /poster -- imageUpload.single('image') middleware, then:
         - If no file, return 400 "No image provided"
         - Call processImage(req.file.buffer, 'poster')
         - Return 201 with { paths: { original, large, medium, thumbnail } }
       - POST /backdrop -- imageUpload.single('image') middleware, then:
         - Call processImage(req.file.buffer, 'backdrop')
         - Return 201 with { paths: { original, large, medium } }
       - Handle Multer errors in an error-handling middleware at the end of the router
       - Export uploadRouter

    2. Create api/src/routes/media.routes.ts:
       - GET /* -- serves files from the uploads directory
       - Resolve the request path against UPLOAD_DIR to prevent path traversal (use path.resolve and verify it starts with UPLOAD_DIR)
       - If file exists, serve it with appropriate Content-Type header (image/webp for .webp files)
       - Set Cache-Control: public, max-age=86400 (1 day for dev)
       - If file not found, return 404
       - This route is PUBLIC (no auth required) -- images need to be visible in the admin panel and eventually the client
       - Use fs.createReadStream for efficient serving (not fs.readFile)
       - Export mediaRouter

    3. Update api/src/routes/index.ts:
       - Import uploadRouter and mediaRouter
       - Register: `app.use("/api/admin/upload", uploadRouter)`
       - Register: `app.use("/api/media", mediaRouter)` -- public route, no admin guard

    4. Call ensureUploadDirs() in the route registration function or at app startup to guarantee directories exist.

    IMPORTANT: Add 'uploads/' to api/.gitignore if not already there. Uploaded files should not be committed.
  </action>

    - `curl -X POST -F "image=@test.txt" http://localhost:5000/api/admin/upload/poster` -- returns 400 (invalid type)
    - Verify uploads/ directory has files in correct subdirectories
