  <action>
    1. Create admin/src/api/content.ts:
       - `listContent(params)`: GET /api/admin/content with query params (type, status, category, search, page, limit, sortBy, sortOrder). Returns { data, total, page, totalPages }.
       - `getContent(id)`: GET /api/admin/content/:id
       - `createContent(data)`: POST /api/admin/content
       - `updateContent(id, data)`: PUT /api/admin/content/:id
       - `deleteContent(id)`: DELETE /api/admin/content/:id
       - `publishContent(id)`: PATCH /api/admin/content/:id/publish
       - `unpublishContent(id)`: PATCH /api/admin/content/:id/unpublish
       - All use apiClient from client.ts
       - Define TypeScript types for Content, ContentListResponse, ContentFilters

    2. Create admin/src/api/categories.ts:
       - `listCategories()`: GET /api/admin/categories
       - `createCategory(name)`: POST /api/admin/categories
       - `updateCategory(id, name)`: PUT /api/admin/categories/:id
       - `deleteCategory(id)`: DELETE /api/admin/categories/:id
       - Define Category type

    3. Create admin/src/hooks/useContentFilters.ts:
       - Custom hook that manages filter state: status ('all'|'published'|'draft'), category (string|undefined), search (string), page (number), sortBy, sortOrder, viewMode ('grid'|'table')
       - Uses URL search params (TanStack Router) to persist filters in URL
       - Exports the filter state and setter functions
       - Debounce search input (300ms) before triggering API call

    4. Create shared components:

       StatusBadge.tsx:
       - Badge showing "Published" (green) or "Draft" (gray/yellow)
       - Uses shadcn Badge component

       FilterBar.tsx:
       - Row with: status pill buttons [All] [Published] [Draft] (active pill highlighted), category dropdown (populated from useQuery for categories), search Input with search icon, view toggle (grid/table icons)
       - Receives filter state and setters as props
       - Category dropdown shows "All Categories" as default option plus all categories from API

       Pagination.tsx:
       - Page info ("Showing 1-20 of 45"), Previous/Next buttons, page number display
       - Receives page, totalPages, onPageChange as props
       - Uses shadcn Button components

       ContentCard.tsx:
       - Card showing poster image (posterPortrait via /api/media/{path}), title, year, quality badge, status badge
       - On hover: semi-transparent overlay with Edit and Delete buttons (use lucide Pencil and Trash2 icons)
       - For channels: show "LIVE" indicator instead of quality
       - Props: content object, onEdit callback, onDelete callback
       - If no poster image, show a placeholder with the content title

       ContentGrid.tsx:
       - CSS grid of ContentCard components
       - Responsive: 5 cols on xl, 4 on lg, 3 on md, 2 on sm
       - Empty state: "No content found" message

       ContentTable.tsx:
       - TanStack Table rendering content in tabular format
       - Uses columns from columns.tsx
       - Sortable columns (click header to toggle sort)
       - Receives data array and sort state from parent

       columns.tsx:
       - Column definitions for TanStack Table:
         - Poster thumbnail (small image), Title, Year, Quality, Status (badge), Categories (comma-joined), Added date (formatted with date-fns)
         - Actions column: Edit and Delete buttons in a DropdownMenu
       - Export separate column defs for movies, documentaries, channels (channels have streamUrl instead of quality)

    Install any missing shadcn components needed (table, dropdown-menu, etc.):
    Run from admin/: `npx shadcn@latest add table select popover command`
