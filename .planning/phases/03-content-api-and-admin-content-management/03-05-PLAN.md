Create Season and Episode CRUD API for managing series hierarchically: Series -> Seasons -> Episodes.

Purpose: Series management requires nested CRUD -- admins create a series (via content API), then add seasons to it, then add episodes to seasons. This plan provides the backend for that hierarchy.
Output: Season CRUD at /api/admin/content/:contentId/seasons, Episode CRUD at /api/admin/content/:contentId/seasons/:seasonId/episodes, all admin-protected.
</objective>

  <action>
    1. Create api/src/validators/season.validators.ts:
       - `createSeasonSchema`: z.object({ number: z.coerce.number().int().positive(), title: z.string().max(255).optional() })
       - `updateSeasonSchema`: z.object({ number: z.coerce.number().int().positive().optional(), title: z.string().max(255).optional() })
       - `createEpisodeSchema`: z.object({
           number: z.coerce.number().int().positive(),
           title: z.string().min(1).max(255).trim(),
           description: z.string().optional(),
           duration: z.coerce.number().int().positive().optional(),
           videoUrl: z.string().optional(),
           thumbnail: z.string().optional()
         })
       - `updateEpisodeSchema`: createEpisodeSchema.partial()

    2. Create api/src/services/season.service.ts:
       Seasons:
       - `listSeasons(contentId)`: findMany where contentId, ordered by number asc, include _count of episodes
       - `createSeason(contentId, data)`: verify content exists and is type SERIES (throw 400 if not), then create season. Handle P2002 unique violation (duplicate season number) with friendly 409 error.
       - `updateSeason(id, data)`: update, handle P2002
       - `deleteSeason(id)`: delete (cascades to episodes)

       Episodes:
       - `listEpisodes(seasonId)`: findMany where seasonId, ordered by number asc
       - `createEpisode(seasonId, data)`: verify season exists, create episode. Handle P2002 (duplicate episode number).
       - `updateEpisode(id, data)`: update, handle P2002
       - `deleteEpisode(id)`: delete

       All functions import prisma from config/database.js.

    3. Create api/src/routes/season.routes.ts:
       Use Express Router with mergeParams: true to access :contentId and :seasonId from parent routes.

       Apply requireAuth, requireAdmin to all routes.

       Season routes (mounted at /api/admin/content/:contentId/seasons):
       - GET / -- list seasons for content
       - POST / -- create season
       - PUT /:seasonId -- update season
       - DELETE /:seasonId -- delete season

       Episode routes (nested under seasons):
       - GET /:seasonId/episodes -- list episodes for season
       - POST /:seasonId/episodes -- create episode
       - PUT /:seasonId/episodes/:episodeId -- update episode
       - DELETE /:seasonId/episodes/:episodeId -- delete episode

       Export seasonRouter.

    4. Update api/src/routes/index.ts:
       - Import seasonRouter
       - Register: `app.use("/api/admin/content/:contentId/seasons", seasonRouter)`

    ESM RULES: .js extensions on all imports.
