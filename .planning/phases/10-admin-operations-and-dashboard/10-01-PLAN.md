```
model SystemSetting {
  key       String   @id
  value     Json
  updatedAt DateTime @updatedAt
  updatedBy String?
}
```

2. Run migration: `cd api && npx prisma migrate dev --name add-system-setting` (if non-interactive fails, use `npx prisma db push` then create migration manually with `npx prisma migrate deploy`).

3. Create api/src/services/dashboard.service.ts:
   - `getDashboardStats(periodDays: number)` -- returns { revenue, activeUsers, totalContent, failedPayments } each with { current, previous, change } using Promise.all. Revenue sums Payment.amount where status="SUCCESS". Active users counts users with sessions active in period. Total content counts Content. Failed payments counts Payment where status="FAILED".
   - `getRevenueChart(months: number)` -- returns Array<{ month: string, revenue: number }> grouped by month. Use raw SQL or Prisma groupBy with date truncation. Return last N months.
   - `getContentBreakdown()` -- returns Array<{ type: string, count: number }> using prisma.content.groupBy({ by: ["type"], _count: true }).
   - `getRecentActivity(params: { period?: string, startDate?: Date, endDate?: Date, page: number, limit: number })` -- queries ActivityLog with date filtering and pagination. Include user relation (select name, email). Period options: "week" (7d), "month" (30d), "quarter" (90d), "all", "custom" (uses startDate/endDate).

4. Create api/src/services/admin-user.service.ts:
   - `listUsers(params)` -- search by name/email/phone, filter by status (active/inactive), sortBy, sortOrder, page, limit. Return users with _count of sessions and payments. Exclude passwordHash from select.
   - `getUser(id)` -- full user details with session list and payment count.
   - `createUser(data)` -- create user with argon2 password hash. Fields: name, email, phone, password, role.
   - `updateUser(id, data)` -- update user fields. If password provided, hash it. Fields: name, email, phone, password, role, isActive.
   - `deleteUser(id)` -- soft delete by setting isActive=false, or hard delete. Use soft delete (set isActive=false).
   - `listUserSessions(userId)` -- returns active sessions for a user.
   - `deleteUserSession(userId, sessionId)` -- deletes a specific session.
   - `exportUsers(params)` -- like listUsers but no pagination (limit 10000), returns all matching users for CSV export.
   - Import argon2 for password hashing. Import PrismaClient from "../generated/prisma/client.js".

5. Create api/src/services/admin-billing.service.ts:
   - `getBillingStats(periodDays: number)` -- returns { totalRevenue, successfulPayments, failedPayments, pendingPayments } with current/previous/change.
   - `listPayments(params)` -- filter by status, userId, search (user name/email), dateRange, page, limit, sortBy, sortOrder. Include user (name, email) and plan (name) relations.
   - `exportPayments(params)` -- like listPayments but no pagination (limit 10000) for CSV export.

6. Create api/src/services/settings.service.ts:
   - `getSetting<T>(key: string, defaultValue: T)` -- findUnique by key, return value or default.
   - `getSettings(keys: string[])` -- findMany, return as Record<string, unknown>.
   - `setSetting(key, value, adminId)` -- upsert with updatedBy.
   - `setSettings(settings: Record<string, unknown>, adminId)` -- batch upsert in transaction.
   - `testMpesaConnection()` -- imports getMpesaClient, attempts OAuth token fetch. Returns { success: boolean, message: string }. For mock environment, return success immediately.
   - Known setting keys: "mpesa.consumerKey", "mpesa.consumerSecret", "mpesa.shortcode", "mpesa.passkey", "mpesa.callbackUrl", "mpesa.environment", "general.siteName", "general.supportEmail", "pricing.weeklyPrice", "pricing.monthlyPrice", "pricing.quarterlyPrice", "limits.maxDevices", "limits.maxInviteCodes".
   - For sensitive keys (containing "key", "secret", "passkey"), mask on read: show only last 4 chars, prefix with "****".

7. Create api/src/services/activity-log.service.ts:
   - `logActivity(params: { userId: string, action: string, entityType: string, entityId?: string, details?: Record<string, unknown>, ipAddress?: string })` -- creates ActivityLog entry. Fire-and-forget safe (catch errors, log to console).
   - `listActivityLogs(params)` -- filter by action, entityType, userId, dateRange, page, limit, sortBy, sortOrder. Include user relation (name, email).

All services use ESM imports with .js extensions. PrismaClient from "../generated/prisma/client.js".
